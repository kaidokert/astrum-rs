# Split-ISR Interrupt Routing — Design Specification

## 1. Overview

The kernel uses a **split-ISR** model to route hardware device interrupts
to partitions. A single generated dispatch handler (`__irq_dispatch`)
serves all bound IRQs, reading the IPSR to identify the source and
looking up the target partition in a static binding table. The kernel
top-half runs in Handler mode (privileged); the partition bottom-half
runs in its next Thread-mode time slot.

See also: `notes/driver-architecture.md` §2 for the broader driver model.

## 2. Four-Step IRQ Flow

1. **IRQ fires** → NVIC vectors into `__irq_dispatch` (Handler mode).
2. **Dispatch handler** reads IPSR, looks up `IrqBinding`, then branches
   on `clear_model`:
   - `PartitionAcks` → masks the IRQ in NVIC to prevent re-fire.
   - `KernelClears(strategy)` → writes an MMIO register to clear the
     hardware source immediately.
3. **Signal partition** via `signal_partition_from_isr` → sets event
   bits, transitions partition from `Waiting` to `Ready`, pends PendSV.
4. **Partition bottom-half** runs in its next time slot, calls
   `event_wait(mask)` or `isr_ring.pop_with(|tag, data| ..)`.

All dispatch paths are data-only (no function pointers), guaranteeing
bounded WCET.

## 3. Data Structures (`kernel/src/irq_dispatch.rs`)

- **`IrqBinding`** (line 103) — `{ irq_num: u8, partition_id: u8,
  event_bits: u32, clear_model: IrqClearModel }`. Constructors:
  `::new(irq, pid, bits)` (defaults to `PartitionAcks`),
  `::with_clear_model(irq, pid, bits, model)`.
- **`IrqClearModel`** (line 148) — `PartitionAcks` | `KernelClears(ClearStrategy)`.
- **`ClearStrategy`** (line 139) — `WriteRegister { addr, value }` |
  `ClearBit { addr, bit }`. `ClearBit` computes
  `1u32.wrapping_shl(bit)` at dispatch time.

## 4. `bind_interrupts!` Macro

Defined in `kernel/src/macros.rs:280`. Generates: `__IRQ_BINDINGS`
(static array), `__IRQ_DIRECT_TABLE` (const `[u8; count]` mapping
each IRQ number to its `__IRQ_BINDINGS` index, or `0xFF` for unbound
slots — built at compile time by `build_direct_table`),
`__irq_dispatch` (handler), `__INTERRUPTS` (IVT in
`.vector_table.interrupts`), `enable_bound_irqs`, `disable_bound_irqs`.

**2-tuple form** (defaults to `PartitionAcks`):

```rust
kernel::bind_interrupts!(MyConfig, 70,
    5  => (0, 0x01),
    10 => (1, 0x02),
);
```

**3-tuple form** (explicit clear model):

```rust
kernel::bind_interrupts!(MyConfig, 70,
    5  => (0, 0x01, IrqClearModel::KernelClears(
        ClearStrategy::WriteRegister { addr: 0x100, value: 1 },
    )),
);
```

The helper `__make_irq_binding!` (`kernel/src/macros.rs:245`) routes
the two forms. Three compile-time assertions reject duplicate IRQ
numbers, out-of-range partition IDs, and IRQ numbers ≥ count.

## 5. Dispatch Handler Operation

Generated at `kernel/src/macros.rs:322`:

1. `mrs {}, ipsr` → extract exception number.
2. `irq_num = (ipsr & 0x1FF) - 16` (external IRQs start at 16).
3. `__IRQ_DIRECT_TABLE.get(irq_num)` — O(1) direct-indexed lookup
   into the compile-time table. Returns the `__IRQ_BINDINGS` index
   for bound IRQs, or `0xFF` for unbound slots (early return).
4. Match `clear_model`:
   - `PartitionAcks` → `NVIC::mask(IrqNr(irq_num))`.
   - `KernelClears` → `write_volatile` to the specified MMIO address.
5. `signal_partition_from_isr::<Config>(partition_id, event_bits)`
   (`kernel/src/irq_dispatch.rs:44`).

`signal_partition_from_isr` acquires a critical section, calls
`signal_partition_inner` to OR event bits and wake the partition,
then pends PendSV if the partition transitioned to `Ready`.

## 6. IsrRingBuffer

Defined in `kernel/src/split_isr.rs`. Used for data-carrying interrupts
(e.g. UART RX bytes).

```rust
pub struct IsrRingBuffer<const D: usize, const M: usize>
```

- `D` — max pending records, `M` — max payload bytes per record.
- `push_from_isr(tag: u8, data: &[u8])` — O(1), returns
  `Err(RingBufferFull)` when full, truncates data to `M` bytes.
- `pop_with(f: FnOnce(u8, &[u8]))` — pops oldest record in-place.
- All storage inline, no heap, single-core Cortex-M safe.

## 7. NVIC Helpers

Generated by `bind_interrupts!` (`kernel/src/macros.rs:378`):

- **`enable_bound_irqs(nvic, priority)`** — sets NVIC priority and
  unmasks each bound IRQ. Called at boot.
- **`disable_bound_irqs()`** — masks each bound IRQ.

Both expand to per-IRQ operations over the binding list. Non-ARM
targets get no-op stubs for `cargo check` compatibility.

`IrqNr(u8)` (`kernel/src/irq_dispatch.rs:89`) implements
`InterruptNumber` on ARM, allowing any raw IRQ number with NVIC APIs.

## 8. Known Limitations

- **`SYS_IRQ_ACK` syscall is implemented** (`kernel/src/irq_ack.rs`,
  `kernel/src/svc.rs` dispatch arm `SyscallId::IrqAck`). Partitions
  using `PartitionAcks` call `SYS_IRQ_ACK` to re-enable their masked
  IRQ after acknowledging the hardware source. The syscall validates
  ownership and clear-model before calling `NVIC::unmask`.
- `lookup_binding` (O(n) linear scan) is retained only for the cold
  `SYS_IRQ_ACK` validation path in `irq_ack.rs`; the hot dispatch
  path uses the O(1) `__IRQ_DIRECT_TABLE`.
- `IsrRingBuffer` is single-producer/single-consumer on single-core
  only; no atomic fencing for multi-core.

## 9. Per-Kernel Binding Table (`Kernel.irq_bindings`)

The `SYS_IRQ_ACK` dispatch path needs access to the binding table
generated by `bind_interrupts!`. The kernel stores the table as a
plain field on the `Kernel` struct, set once at boot via
`store_irq_bindings()`.

### 9.1 Kernel Field

```rust
pub struct Kernel<C: KernelConfig> {
    // ...
    pub irq_bindings: &'static [IrqBinding],
}
```

- **`irq_bindings`** — a `&'static` slice referencing the
  `__IRQ_BINDINGS` array generated by `bind_interrupts!`.
  Initialised to `&[]` (empty) at kernel construction; populated
  by `store_irq_bindings()` during `enable_bound_irqs`.

### 9.2 `store_irq_bindings` Semantics

```rust
pub fn store_irq_bindings(&mut self, bindings: &'static [IrqBinding]) {
    self.irq_bindings = bindings;
}
```

Simple assignment — overwrites the slice reference. Called exactly
once during boot inside `enable_bound_irqs`. The `&mut self`
receiver guarantees exclusive access; no return value.

### 9.3 No Atomics Required

No atomic operations or lock-free registration are needed because:

- **Single-core Cortex-M** — no parallel writers.
- **Called under `with_kernel_mut`** — wraps the call in
  `cortex_m::interrupt::free()`, masking all configurable
  interrupts for the duration.
- **Plain `&mut self`** — Rust's borrow checker enforces exclusive
  access at compile time within the closure.

### 9.4 Boot-Time Wiring

`enable_bound_irqs` (generated by `bind_interrupts!` in
`kernel/src/macros.rs`) performs two steps:

1. **Store bindings** inside a `with_kernel_mut` critical section:
   ```rust
   crate::state::with_kernel_mut::<Config, _, _>(|k| {
       k.store_irq_bindings(&__IRQ_BINDINGS);
   });
   ```
2. **Enable each IRQ** with per-IRQ `set_priority` + `unmask`:
   ```rust
   // SAFETY: NVIC priority/unmask calls require that the IRQ number is valid
   // and that no data race exists on the NVIC registers. Both are guaranteed:
   // IRQ numbers are compile-time validated by bind_interrupts!, and this runs
   // single-threaded at boot before any IRQ is enabled.
   unsafe {
       nvic.set_priority(IrqNr(irq), priority);
       NVIC::unmask(IrqNr(irq));
   }
   ```

The store happens before any IRQ is unmasked, ensuring the binding
table is populated before any IRQ can fire and before any partition
can issue `SYS_IRQ_ACK`.

### 9.5 QEMU Integration Tests

## 10. Implementation Status

All split-ISR interrupt routing primitives are implemented and tested.

### 10.1 Custom-Handler Binding (`handler:` form)

The `bind_interrupts!` macro supports a `handler:` variant that routes
an IRQ to a custom ISR function instead of the default `__irq_dispatch`
handler. This is the recommended pattern for driver ISRs that need to
perform device-specific work (e.g. reading a FIFO) before signalling
the partition.

**Syntax** — the `handler:` keyword replaces the clear-model position
in the 3-tuple form:

```rust
// SAFETY: This function is only called by the NVIC as an interrupt handler.
// It runs in Handler mode with access to kernel statics and must complete
// before returning to the interrupted context.
unsafe extern "C" fn uart_rx_isr() {
    // Read UART DR, push into IsrRingBuffer, signal partition.
}

kernel::bind_interrupts!(AppConfig, 90,
    11 => (0, 0x01, handler: uart_rx_isr),
);
```

Standard and `handler:` bindings can be mixed in a single invocation:

```rust
kernel::bind_interrupts!(AppConfig, 100,
    15 => (0, 0x01),                          // default dispatch
    25 => (1, 0x04, handler: uart_rx_isr),    // custom ISR
    35 => (0, 0x10),                          // default dispatch
);
```

**Semantics:**

- `__make_irq_binding!` still creates an `IrqBinding::new(irq, pid, evt)`
  entry (defaults to `PartitionAcks`) so `SYS_IRQ_ACK` ownership
  validation works for the custom handler's IRQ.
- `__make_irq_handler!` places the custom function pointer in the IVT
  slot instead of `__irq_dispatch`.
- The custom ISR is responsible for its own clear/mask logic and for
  calling `signal_partition_from_isr` if it needs to wake a partition.

### 10.2 Overflow Counter Observability

`IsrRingBuffer` tracks dropped pushes via a saturating counter
(`kernel/src/split_isr.rs`):

- **`push_from_isr(&mut self, tag: u8, data: &[u8]) -> Result<(), RingBufferFull>`** —
  on failure, increments the internal overflow counter (saturating)
  and returns `Err(RingBufferFull)`.
- **`overflow_count(&self) -> usize`** — returns total drops since
  last reset.
- **`reset_overflow_count(&mut self) -> usize`** — returns the current
  count and resets it to zero.

**Synchronization — primary trade-off:** All mutating methods take
`&mut self`; the buffer is *not* lock-free.  In practice this means
**every `pop` / `reset_overflow_count` call must disable interrupts**
(or mask the specific IRQ) to prevent the ISR from calling
`push_from_isr` concurrently.  On single-core Cortex-M the two options
are:

1. **Global critical section** — `cortex_m::interrupt::free(|| …)`.
   Simple but blocks *all* interrupts for the duration of the drain
   loop, increasing worst-case interrupt latency.
2. **Per-IRQ masking** — `NVIC::mask(irq)` before draining, unmask
   after.  Narrower blast radius but requires the caller to know
   which IRQ feeds the buffer.

Usage pattern for partition-side diagnostics:

```rust
// Drain all pending events.
while ring.pop_with(|tag, data| { /* process */ }) {}

// Check for back-pressure since last drain.
let dropped = ring.reset_overflow_count();
if dropped > 0 {
    log_overflow(dropped);
}
```

### 10.3 Compile-Time Validation

Five `const` assertions fire at build time inside the `bind_interrupts!`
macro (`kernel/src/macros.rs`, line 308). All run on every target
(including host `cargo check`), not just ARM:

1. **Max IRQ count** — `count ≤ 240`, matching the Cortex-M3 maximum
   external interrupt count.
   Error: `"bind_interrupts!: count exceeds Cortex-M3 maximum (240)"`
2. **No duplicate IRQ numbers** — O(n²) pairwise comparison via
   `irq_dispatch::has_duplicate_irqs`.
   Error: `"bind_interrupts!: duplicate IRQ number"`
3. **Partition ID in range** — each `partition_id < KernelConfig::N`
   via `irq_dispatch::has_invalid_partition_id`.
   Error: `"bind_interrupts!: partition_id out of range"`
4. **Non-zero event bits** — rejects any binding where `event_bits == 0`,
   which would be a silent no-op at dispatch time, via
   `irq_dispatch::has_zero_event_bits`.
   Error: `"bind_interrupts!: event_bits == 0 is a no-op"`
5. **IRQ number within table bounds** — per-binding `irq < count`.
   Error: `"bind_interrupts!: IRQ number >= count"`

Additionally, a `compile_error!` fires on ARM (non-test) builds when
the `custom-ivt` Cargo feature is not enabled, since `bind_interrupts!`
emits a `#[link_section = ".vector_table.interrupts"]` static that
conflicts with `cortex-m-rt`'s default IVT.
