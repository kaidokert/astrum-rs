# SysTick Priority Policy: Four-Tier Exception Model

The kernel enforces a four-tier Cortex-M exception priority ordering.
Lower numeric value = higher urgency (preempts more).

| Tier | Exception   | Default | Purpose                         |
|------|-------------|---------|---------------------------------|
| 1    | SVCall      | `0x00`  | Syscall entry — never preempted |
| 2    | Device IRQs | `0xC0`  | Split-ISR top-half handlers     |
| 3    | SysTick     | `0xFE`  | Partition time-slot tick        |
| 4    | PendSV      | `0xFF`  | Deferred context switch         |

## Configuration (`KernelConfig` trait, `kernel/src/config.rs`)

Constants: `SVCALL_PRIORITY` (0x00), `IRQ_DEFAULT_PRIORITY` (0xC0),
`SYSTICK_PRIORITY` (0xFE), `PENDSV_PRIORITY` (0xFF). Implementations may
override values if the ordering invariant holds.

## Compile-Time Enforcement: `assert_priority_order`

`assert_priority_order::<C>()` is a `const fn` that statically verifies
`SVCALL < IRQ_DEFAULT < SYSTICK < PENDSV` for any `KernelConfig`. A
mis-ordered custom config is a compile-time error.

## Runtime Wiring: `enable_bound_irqs`

Generated by `bind_interrupts!`: registers the IRQ binding table, sets each
IRQ's NVIC priority to `Config::IRQ_DEFAULT_PRIORITY`, and unmasks it.

## Design Rationale

**SVCall highest (0x00):** No exception may preempt a syscall or it could
corrupt kernel state.
**Device IRQs above SysTick:** Split-ISR top-halves must drain hardware FIFOs
into `IsrRingBuffer` with minimal latency — SysTick preemption would add
unacceptable jitter for hard-realtime peripherals.

**SysTick above PendSV:** SysTick advances the schedule and may pend a context
switch. PendSV must not preempt mid-update or it switches on stale state.

**PendSV lowest (0xFF):** Deferred context switch runs only after all
higher-priority work completes — standard ARM Cortex-M pattern.
